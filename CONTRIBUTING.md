# Contributing to vetted-field-connect

Thanks for your interest in contributing. This project powers a worker vetting and placement platform, so we hold contributors to the same standard of trust we expect of the workers on the platform.

## Start Here

1. Read [`ONBOARDING.md`](./ONBOARDING.md) — it covers vetting requirements, the Code of Conduct, and contribution guidelines.
2. Read [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md).
3. Find a maintainer contact in [`MAINTAINERS.md`](./MAINTAINERS.md).

## Required Vetting (summary)

Before any pull request can be merged, you must privately submit:
- **3 professional references**, one for each of your last 3 jobs, all **completed within the last 6 months**.
- **Project photos** for each of those 3 jobs (minimum 3 photos per job, 9 total) with captions and before/after where applicable.

Send these to the private channel listed in [`MAINTAINERS.md`](./MAINTAINERS.md). **Do not** post references, contact info, or photos in PRs, issues, or commits.

## Workflow

1. Open an issue describing the bug or feature before writing significant code.
2. Fork the repo and create a feature branch (`feat/...`, `fix/...`, `docs/...`, `chore/...`).
3. Open your PR as a **draft** while vetting is in progress. The PR template will populate the checklist for you.
4. Keep PRs focused, rebased on `main`, and passing CI.
5. A maintainer reviews after vetting is complete.

## Reporting Issues

- Bugs and feature requests → use the GitHub issue templates.
- Security vulnerabilities → email **jkservices091@gmail.com** with subject `[Security]`. Do not open a public issue.
- Code of Conduct concerns → email **jkservices091@gmail.com** with subject `[Conduct]`.
